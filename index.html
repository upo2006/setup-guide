1

Supabase アカウント作成

Supabase は無料で使えるデータベースサービスです。
アカウントをお持ちでない方は、まず作成してください。

①

下のボタンから Supabase を開く

ログイン画面が表示されます

🔗 Supabase を開く
②

画面下部の「Don't have an account? Sign up」をクリック

※「Sign in」を押すとログイン画面になります。Sign up から進んでください

③

メールアドレスと希望のパスワードを入力後、「Sign up」をクリック

パスワードの要件：

✅ 大文字を含む（A〜Z）

✅ 小文字を含む（a〜z）

✅ 数字を含む（0〜9）

✅ 記号を含む（例：!?<>@#$%）

✅ 8文字以上

💡 例：Soccer2024!

④

メール認証を完了する

Supabase から「メールアドレスを確認して、Supabaseでプロジェクトを開始しましょう！」というメールが届きます

メール内の緑のボタン「メールアドレスを確認」を押してください

⑤

「Create a new organization」画面が表示されます

・Name：チーム名を入力（後から変更可能）

・Type：Personal のまま

・Plan：Free - $0/month のまま

「Create organization」をクリック

✅ アカウント作成が完了したら「次へ」に進んでください

2

プロジェクト作成

「Create a new project」画面が表示されます。以下の通りに入力してください。

①

Organization：そのまま

②

GitHub (optional)：無視でOK

③

Project name：チーム名など分かりやすい名前に変更

例：my-team（半角英数字・ハイフン）

④

Database password：強いパスワードを入力

「Generate a password」をクリックすると自動生成できます

⚠️ 生成した場合は必ずメモしてください

⑤

Region：Asia-Pacific のまま

⑥

Security：そのまま

⑦

「Create new project」をクリック

作成に1〜2分かかります

🔑 プロジェクト名の下にURLが表示されています

横の「Copy」ボタンを押すとメニューが出ます。
「Project URL」を選んでコピーしてください

例：https://xxxxxxxxxx.supabase.co

✅ URLを入力したら「次へ」に進んでください

3

APIキーの取得

アプリのログインに使う anon キーを取得します。

1

左メニュー一番下の「Project Settings」をクリック

2

左メニューの「API Keys」をクリック

※「JWT Keys」ではなく「API Keys」です

3

上部タブの「Legacy anon, service_role API keys」をクリック

🗝️

「anon / public」のキーをコピー

「eyJ...」から始まる長いキー・右側「Copy」をクリック

⚠️ 「service_role / secret」は使用しない
「anon / public」のキーだけを使ってください

🗝️ コピーしたキーをここに入力・保存

※ この情報はこのデバイス内にのみ保存されます

4

テーブル作成（SQL実行）

データベースにアプリが使うテーブルを作成します。
※ AMS・MRS 共通のSQLです

①

左メニューの「SQL Editor」を開く

②

下の「📋 コピー」ボタンでSQLをコピー

③

画面中央の黒い入力エリアをクリックして貼り付け

「hit CTRL+SHIFT+K...」と書いてある部分です

④

右上の「Run」ボタンをクリック

Ctrl+Enter でも実行できます

✅ 画面下部に「Success. No rows returned」と表示されれば成功です

setup.sql（AMS・MRS 共通） 
-- ① イベントテーブル（AMS用）
CREATE TABLE IF NOT EXISTS attendance_events (
  id          BIGSERIAL PRIMARY KEY,
  title       TEXT NOT NULL,
  event_date  DATE NOT NULL,
  event_time  TEXT,
  category    TEXT,
  location    TEXT,
  transport   TEXT,
  memo        TEXT,
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- ② 出欠回答テーブル（AMS用）
CREATE TABLE IF NOT EXISTS attendance_responses (
  id             BIGSERIAL PRIMARY KEY,
  event_id       BIGINT REFERENCES attendance_events(id) ON DELETE CASCADE,
  jersey_number  TEXT NOT NULL,
  attendance     TEXT,
  transport_need TEXT,
  carpool        TEXT,
  notes          TEXT,
  created_at     TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(event_id, jersey_number)
);

-- ③ 選手・データ管理テーブル（AMS・MRS 共通）
CREATE TABLE IF NOT EXISTS app_records (
  id         BIGSERIAL PRIMARY KEY,
  type       TEXT NOT NULL,
  payload    JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- ④ セキュリティポリシー（Row Level Security）
ALTER TABLE attendance_events    ENABLE ROW LEVEL SECURITY;
ALTER TABLE attendance_responses ENABLE ROW LEVEL SECURITY;
ALTER TABLE app_records          ENABLE ROW LEVEL SECURITY;

-- 全ユーザーの読み取りを許可
CREATE POLICY "public_select_events"    ON attendance_events   FOR SELECT USING (true);
CREATE POLICY "public_select_responses" ON attendance_responses FOR SELECT USING (true);
CREATE POLICY "public_select_records"   ON app_records         FOR SELECT USING (true);

-- 全ユーザーの書き込みを許可（出欠回答）
CREATE POLICY "public_insert_responses" ON attendance_responses FOR INSERT WITH CHECK (true);
CREATE POLICY "public_update_responses" ON attendance_responses FOR UPDATE USING (true);

-- 認証ユーザーのみ イベント・選手を管理
CREATE POLICY "auth_all_events"  ON attendance_events FOR ALL TO authenticated USING (true) WITH CHECK (true);
CREATE POLICY "auth_all_records" ON app_records       FOR ALL TO authenticated USING (true) WITH CHECK (true);

✅ 「Success. No rows returned」と表示されれば完了です

5

管理者ユーザーの作成

アプリにログインするための管理者アカウントを作成します。

①

左メニューの「Authentication」をクリック

🔒 のアイコンが目印です

②

上部タブの「Users」をクリック

ユーザー一覧が表示されます

③

右上の「Add user」→「Create new user」を選択

④

メールアドレスとパスワードを入力

これがアプリへのログイン情報になります

※「Auto Confirm User?」は ON のまま

⑤

「Create user」ボタンをクリック

ユーザー一覧に追加されれば完了です

💡 入力したメールアドレスとパスワードは大切に保管してください。
アプリへのログインに使用します。

①

チーム名：チームの名前

anon public キー：手順3でコピーしたキー

④

「接続テスト＆保存」をクリック

✅ 成功が表示されればセットアップ完了です！
