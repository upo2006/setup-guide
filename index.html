AMS セットアップガイド

Attendance Management System

ステップ 1 / 6
1

Supabase アカウント作成

Supabase は無料で使えるデータベースサービスです。
アカウントをお持ちでない方は、まず作成してください。

①

下のボタンから Supabase を開く

ログイン画面が表示されます

②

画面下部の「Don't have an account? Sign up」をクリック

※「Start your project」を押すとログイン画面になります。Sign up から進んでください

③

メールアドレスとパスワードを入力して登録

④

メール認証を完了する

登録メールに届く確認メールのリンクをクリック

🔗 Supabase を開く

✅ アカウント作成が完了したら「次へ」に進んでください

2

プロジェクト作成

Supabase ダッシュボードで新しいプロジェクトを作成します。

①

「New project」をクリック

ダッシュボード上部または中央にあります

②

プロジェクト名を入力

例：my-team-ams（半角英数字・ハイフン）

③

データベースパスワードを設定

強いパスワードを設定してどこかにメモしてください

④

リージョンを選択

推奨 Northeast Asia (Tokyo)

⑤

「Create new project」をクリック

作成に1〜2分かかります

✅ プロジェクトのダッシュボードが表示されたら「次へ」に進んでください

3

URL・APIキーの取得

AMS の接続に必要な2つの情報を取得します。

①

左メニューの「Project Settings」を開く

歯車アイコン ⚙️ をクリック

②

「Data API」または「API」タブを開く

🔑

Project URL をコピー

例：https://xxxxxxxxxx.supabase.co

→ AMSの「Supabase URL」に貼り付ける

🗝️

anon public キーをコピー

「Project API keys」の中の anon / public キー

→ AMSの「anon public キー」に貼り付ける

⚠️ service_role キーは使用しない
anon / public と書かれているキーだけを使ってください

4

テーブル作成（SQL実行）

データベースにAMSが使うテーブルを作成します。

①

左メニューの「SQL Editor」を開く

②

下のSQLをコピーして貼り付ける

③

「Run」ボタンをクリック

▶ または Ctrl+Enter で実行

setup.sql 
-- ① イベントテーブル
CREATE TABLE IF NOT EXISTS attendance_events (
  id          BIGSERIAL PRIMARY KEY,
  title       TEXT NOT NULL,
  event_date  DATE NOT NULL,
  event_time  TEXT,
  category    TEXT,
  location    TEXT,
  transport   TEXT,
  memo        TEXT,
  created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- ② 出欠回答テーブル
CREATE TABLE IF NOT EXISTS attendance_responses (
  id             BIGSERIAL PRIMARY KEY,
  event_id       BIGINT REFERENCES attendance_events(id) ON DELETE CASCADE,
  jersey_number  TEXT NOT NULL,
  attendance     TEXT,
  transport_need TEXT,
  carpool        TEXT,
  notes          TEXT,
  created_at     TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(event_id, jersey_number)
);

-- ③ 選手・データ管理テーブル
CREATE TABLE IF NOT EXISTS app_records (
  id         BIGSERIAL PRIMARY KEY,
  type       TEXT NOT NULL,
  payload    JSONB,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- ④ セキュリティポリシー（Row Level Security）
ALTER TABLE attendance_events   ENABLE ROW LEVEL SECURITY;
ALTER TABLE attendance_responses ENABLE ROW LEVEL SECURITY;
ALTER TABLE app_records         ENABLE ROW LEVEL SECURITY;

-- 全ユーザーの読み取りを許可
CREATE POLICY "public_select_events"    ON attendance_events   FOR SELECT USING (true);
CREATE POLICY "public_select_responses" ON attendance_responses FOR SELECT USING (true);
CREATE POLICY "public_select_records"   ON app_records         FOR SELECT USING (true);

-- 全ユーザーの書き込みを許可（出欠回答）
CREATE POLICY "public_insert_responses" ON attendance_responses FOR INSERT WITH CHECK (true);
CREATE POLICY "public_update_responses" ON attendance_responses FOR UPDATE USING (true);

-- 認証ユーザーのみ イベント・選手を管理
CREATE POLICY "auth_all_events"   ON attendance_events   FOR ALL TO authenticated USING (true) WITH CHECK (true);
CREATE POLICY "auth_all_records"  ON app_records         FOR ALL TO authenticated USING (true) WITH CHECK (true);

✅ 「Success. No rows returned」と表示されれば完了です

5

管理者ユーザーの作成

AMSにログインするための管理者アカウントを作成します。

①

左メニューの「Authentication」を開く

🔒 アイコンをクリック

②

「Users」タブ →「Add user」をクリック

③

メールアドレスとパスワードを入力

これがAMSへのログイン情報になります

④

「Create user」で作成

💡 メールアドレスとパスワードは大切に保管してください。
AMS へのログインに使用します。

6

AMS に接続設定を入力

ステップ3で取得した URL とキーを AMS に入力します。

①

AMS を開く

upo2006.github.io/ams-app/

②

右上の ⚙️ → 🔑 ログイン

③

接続設定フォームに入力

チーム名：チームの名前

Supabase URL：手順3でコピーしたURL

anon public キー：手順3でコピーしたキー

④

「接続テスト＆保存」をクリック

✅ 成功が表示されればセットアップ完了です！

🎉

セットアップ完了！

AMS でチームの出欠管理を始めましょう
